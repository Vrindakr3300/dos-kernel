**TL;DR** — This is the distribution release: `pip install dos-kernel` now delivers the compiled `dos-hook` fast path inside per-platform wheels, the install surface grows uv/uvx + wrapper + WSL paths gated against doc drift, and a plugin install becomes verifiable against what git actually shipped. Operators also get `dos helped --explain` and a faster truth syscall.

## `pypi` — the native hook binary ships in per-platform wheels (docs/286)

- **`pip install dos-kernel` now bundles the compiled `dos-hook` fast-path binary** — no Go toolchain or plugin clone needed.
  - *Why:* the 16× hook fast path previously reached users only via the marketplace plugin's committed binaries; the wheel is the lower-friction artifact.
  - *How:* `scripts/build_wheels.py` builds ABI-`none` per-platform wheels (one Linux runner cross-compiles the amd64+arm64 × linux/macos/windows matrix); `src/dos/hook_binary.py` locates the bundled binary under `src/dos/_bin/` with a PATH fallback.
  - *Impact:* wheels are per-platform now, not `py3-none-any`; the sdist stays pure-Python and falls back to a PATH-resolved binary when none is bundled.
- **CI builds the full wheel matrix and gates the binary format** — the `build/` staging leak that put a macOS binary in the Windows wheel is fixed.
  - *How:* `.github/workflows/{ci,publish}.yml` + `tests/test_build_wheels_binary_format.py` (inspects the shipped bytes; opt-in locally via `DOS_TEST_BUILD_WHEELS=1`, always on in clean CI).
- **`hook_binary.py` references `DEFAULT_DIALECT` instead of a vendor literal** — restores the vendor-agnostic-kernel litmus on the new module.

## `install` — first-class install options + a drift gate

- **uv/uvx lead the install story, with `install.sh` / `install.ps1` wrappers and a dedicated `docs/INSTALL.md`.**
  - *How:* `scripts/install_facts.py` single-sources extras and console scripts from `pyproject.toml`; `tests/test_install_drift.py` fails any doc naming a missing extra or wrong dist name.
  - *How:* `tests/test_install_levels.py` performs real installs (uvx, uv tool, pip venv, and WSL from Windows), each proven by running `dos doctor`.
- **`install.py` wires the bin dir onto PATH via the shell profile on POSIX**, with clean removal on uninstall.

## `plugin` — installs you can verify

- **Install-integrity verifier** — "is this install what git shipped?"
  - *How:* `scripts/verify_plugin_install.py` compares every installed plugin file against `HEAD:claude-plugin` by git blob SHA.
- **Launcher-reachability check** — a bundled launcher must be reachable from `hooks.json`, not merely present.
  - *Why:* `dos-hook.ps1` sat orphaned (dead code behind a comment that lied); the check makes that class of drift structural.
  - *How:* `scripts/build_plugin.py` reachability audit + `tests/test_plugin_manifest.py` pins.

## `cli` — operator observability (docs/285 Phase 2)

- **`dos helped --explain` glosses each refusal class** so an operator reads *why* the kernel refused, not just how often.
  - *How:* per-reason glosses in `src/dos/help_summary.py`; default `dos helped` output unchanged.

## `oracle` — faster truth syscall

- **The registry collision check rides the touched-file memo** — one batched `git log` instead of a per-pair walk (docs/284 follow-up).
  - *How:* `src/dos/oracle.py` routes collision detection through the existing memo; pinned by `tests/test_oracle_grep_in_process.py`.

## portability + publishing hygiene

- **The suite is proven on Linux/WSL** — 3856 passing, zero kernel defects; four portability fixes landed.
  - *How:* the hermes example no longer hangs on an unattended `sudo` prompt (`stdin=DEVNULL` + timeout); filelock, plugin-manifest, and posttool-sensor tests made cross-platform. Audit reports under `docs/reports/`.
- **A durable leak gate guards published text** — re-accumulated private-path leaks scrubbed once, then gated.
  - *How:* `scripts/leak_scan.py` + a CI step + `.gitignore`/`.env.example` conventions.

## `docs`
