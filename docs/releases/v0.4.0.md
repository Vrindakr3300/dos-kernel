**TL;DR** — DOS gains its fourth distrust syscall: `liveness()` answers "is this
run *moving*, or just spinning?" from ground-truth git + journal evidence — the
temporal sibling of `verify()`. It also gains the **JUDGE rung**: a pluggable,
advisory adjudicator seam for the residue the deterministic oracle abstains on,
hedged so an open adjudicator set stays safe. `verify()` now works out-of-the-box
on any git repo (generic ship-stamp grammar by default), and the package picks up
the adoption surface — install script, quickstart, playbooks — to read as a clean,
public trust substrate.

## liveness() — the temporal distrust verdict (LVN, docs/82)

- **The 4th syscall: `liveness()` classifies a run as ADVANCING / SPINNING /
  STALLED** from a `ProgressEvidence` of commits-since-start + lane-journal deltas
  + the wall clock — never the agent's "making progress" narration.
  - *Why:* `verify()` distrusts a *finished* claim ("I shipped P"); `liveness()`
    distrusts an *in-flight* one ("I'm making progress"). A run can be alive
    (heartbeating) yet not moving — that is SPINNING, and only ground truth shows it.
  - *How:* `src/dos/liveness.py` — a PURE `classify(ProgressEvidence, policy) ->
    LivenessVerdict` (state in, verdict out, no I/O — the arbiter discipline).
- **`dos liveness` — the verdict IS the exit code** (ADVANCING=0, SPINNING=3,
  STALLED=4), so a shell or loop can branch without re-parsing. Evidence is
  gathered at the CLI boundary; works against a plain git repo with no plan,
  registry, or journal (commits-since-start alone suffices — the `verify` no-plan rail).
- **Phase 2 — the lane-journal rung.** `journal_delta.fold_since` is a PURE fold
  of journal entries → events-since-start + newest-beat-age, scoped to a run's
  `(loop_ts, lane)` lease (identity required — no host-wide guess). A HEARTBEAT
  proves life but is *not* progress, which is what makes SPINNING reachable; every
  degrade path (corrupt entry, unparseable stamp, future-dated beat) fails safe.
  - *How:* `src/dos/journal_delta.py` + the shared `src/dos/git_delta.py` commit
    reader (`dos.timeline` reads it too).
- **Advisory.** `liveness()` reports — it never kills a process or refuses a lease.

## The judge rung — pluggable adjudicators (docs/86)

- **A blocked claim now flows through an oversight hierarchy: ORACLE → JUDGE →
  HUMAN.** The ORACLE (kernel `verify()`) is deterministic and forgery-proof but
  *narrow* — it abstains on what it can't mechanically cross-check. The **JUDGE**
  rung (a driver) rules on that residue: a model, a heuristic, a debate, a learned
  verifier. The HUMAN (`dos decisions` queue) is the scarce last resort.
  - *How:* `src/dos/judges.py` holds only the pure `Judge` protocol + verdict +
    `run_judge` + the resolver + the built-in `AbstainJudge`; `drivers/llm_judge`
    is the first *ruling* occupant.
- **What keeps an open adjudicator set safe (four-fold discipline):**
  deterministic-first, advisory-only, **fail-to-abstain** (`run_judge` converts any
  raise/bad-return to ABSTAIN, never AGREE), abstention-first. A judge can never
  *clear* a claim the oracle didn't — the judge analogue of the predicate
  conjunctive-only rule. New litmus in CLAUDE.md ("the kernel never imports a judge
  implementation"), AST-pinned by `tests/test_judges.py` + `test_vendor_agnostic_kernel.py`.
- **A judge eval harness.** `judge_eval.score(judge, cases) -> JudgeReport` makes a
  judge produce a number — chiefly the **false-clear rate**, the dangerous cell
  where a judge waves through a claim that is in fact false. `examples/dos_ext/`
  is a bring-your-own-judge worked example over the entry-point seam.
- **Vendor-agnostic proof.** New pins show the kernel and the judge seam are
  vendor-blind — no identity channel, AST + signature rather than a CLI name — so
  DOS works with Gemini/Codex, not just Claude.

## verify() works on a foreign repo out-of-the-box (docs/83)

- **The generic config now ships a generic ship-stamp grammar.** `default_config`
  carries `GENERIC_STAMP_CONVENTION` — a bare `<SERIES>: <PHASE>` / `<slug> Phase
  <N>:` counts as a direct ship, with no required directory prefix — so `dos verify`
  answers on a plain repo with zero `dos.toml`.
  - *Why:* the SCV strict-default was a reuse blocker; this is the user-approved
    reversal, for the generic path only.
  - *How:* `src/dos/{config,stamp,phase_shipped}.py`. `job_config` keeps the strict
    `(docs|go|…)/` grammar + job's bookkeeping guards, so `job` and the kernel suite
    are byte-unchanged. The generic convention still carries the universal
    release-bundle + bulk-snapshot guards — loosened, not a free-for-all. A repo that
    needs strict grammar declares it in `dos.toml [stamp]` (or passes `--job`).

## FleetHorizon benchmark — velocity + a training-data factory (docs/81, docs/84)

- **A second fleet-effectiveness axis beside integrity: velocity.** Coordination
  dividend, merge-conflict detonation cost, review-queue bottleneck — the
  verified-velocity-per-$ headline flips DOS-positive once the downstream bill is priced.
- **The closed loop as a ground-truth-LABEL factory** (`trajectory.py`,
  `verifier.py`): features ⟂ git-label ⟂ verdict, with a distillation experiment that
  finds the irreducibility boundary (pure lies are learnable from shape; FLAKES are
  not — only git separates them).
- **A heterogeneous-vendor drop-in** (`vendors.py`) proves the benchmark drives
  Gemini/Codex-shaped failure mixes, plus graphs, an opt-in live multi-CLI demo, and
  a pinned results snapshot.
