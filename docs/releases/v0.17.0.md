**TL;DR** — A performance + evidence release. The `dos-mcp` server is now ~3.5x
faster on every tool call (the truth syscall stopped spawning a second Python
interpreter to grep git; the per-call runtime-`EnvPrint` probe is gone). `verify()`
gained a non-git evidence rung so a green-CI signal can sharpen a ship verdict
(conjunctively — it can never manufacture a ship). The Go `dos-hook` fast-path
became observable, the kernel's pure-verdict invariants are now property-tested, and
several worked-example + paper surfaces landed.

## `mcp` / `oracle` — a ~3.5x faster MCP server (docs/275)

- **The truth syscall greps git in-process instead of spawning a Python child.**
  `oracle.default_grep_fallback_batch` used to run `python -m dos.phase_shipped
  --batch`, which re-ran `import dos` (~170 ms of pure interpreter startup) just to
  grep — a tax a long-lived server paid on every `dos_verify`.
  - *Why:* the parent already imports `phase_shipped`; the process boundary was never
    part of the verdict, only a cost.
  - *How:* `src/dos/oracle.py:_grep_batch_in_process` runs the same functions the
    child's `--batch` main ran; the subprocess stays as a fallback behind
    `DOS_ORACLE_GREP_SUBPROCESS=1`. Verdicts are byte-identical.
- **The MCP server skips the per-call `EnvPrint` probe.** No tool reads `cfg.env`, so
  building it (a `git rev-parse` + a platform query) on every call was waste.
  - *How:* `default_config`/`job_config`/`load_workspace_config` take `gather_env`
    (default `True`, so the CLI is unchanged); the server passes `gather_env=False`.
    `gather_env_print()` is also memoized per process, and `lane_journal.JOURNAL_PATH`
    resolves lazily so `import dos` no longer forces a config build.
- **Measured (win32-AMD64):** `dos_verify` ~251 → ~71 ms; pure-verdict tools
  (`dos_arbitrate`/`dos_refuse_reasons`/`dos_doctor`) ~18 → ~3 ms; `import dos`
  ~125 → ~73 ms. Caching env-authored facts (git SHA, platform, commit footprint)
  changes latency, not trust.
- **Pinned:** new `tests/test_oracle_grep_in_process.py` (in-process ≡ subprocess
  equivalence + a structural "no python child spawned" guard), extended env-print /
  lane-journal / MCP tests, and a repeatable harness `scripts/bench_mcp.py` that
  gates on the equivalence.

## `verify` — the non-git evidence rung (docs/265)

- **`verify()` can consult an out-of-kernel witness beyond git** — e.g. CI status —
  and fold it CONJUNCTIVELY onto a git ship verdict.
  - *Why:* a commit that exists in git but failed CI should not read as a clean ship.
  - *How:* GREEN upgrades `source` to `ci-green`; RED withholds the upgrade;
    NO_SIGNAL passes the git verdict through. It can never promote `shipped=False →
    True` — green CI without a reachable commit manufactures nothing (the §1 safety
    invariant). Resolved by name via the `dos.evidence_sources` group
    (`CiStatusSource`); the kernel names no provider.

## `dos-hook` — Go-kernel observability (docs/276)

- **The native Go `dos-hook` counts every verdict and surfaces it.** `dos-hook stats`
  reports the verdict tallies, with a `--since` window to scope the view.
  - *How:* `go/internal/hook/stats.go` + `go/cmd/dos-hook/main.go`; byte-exact with
    the Python sensors, the fast-path stays ~tens-of-× faster than a Python hook call.

## `scout` — outcome-driven replan (rule-10)

- **`LaneOutcomeShape` slice + outcome-driven replan prompts.** A lane's outcome
  shape (a runs FLOOR, `is_actionable`) feeds a rule-10 replan suggestion.
