**TL;DR** — This release ports the hook fast-path into a native Go `dos-hook` binary that serves the pretool/posttool/marker/stop verbs byte-for-byte identically to the Python sensors but ~60x faster, and adds two new kernel verdicts: `efficiency` (did the tokens buy work?) and the `verdict_journal` + `dos observe` observability surface. It also generalizes the wait-marker budget (`noop_streak`), adds the `dos-goal-gate` skill and two new benchmarks, and lands a paper/README rewrite. No breaking ABI change — `verify`/`arbitrate`/`refuse` signatures and the verdict vocabularies are unchanged, only added to.

## `go-hook` — a native `dos-hook` fast-path (docs/125)

- **`dos-hook` serves `pretool`, `posttool`, `marker`, and `stop` natively in Go** — byte-exact with the Python sensors, ~60x faster on the hot path.
  - *Why:* the hook runs on every tool call; a Python cold-start per call is the dominant sensing latency, and latency bounds loop gain.
  - *How:* `go/cmd/dos-hook/main.go` dispatches; `go/internal/hook/{admission,decide,posttool,marker,stop,verify,overlap,wal,workspace,selfmodify}.go`. Parity is pinned against a generated corpus per verb (`go/internal/hook/parity/corpus_*.jsonl`, `parity_*_test.go`) and cross-checked by `tests/test_go_hook_parity.py`.
- **Cross-compile build + `bin/` launchers + native-default flip** — the plugin ships platform launchers and prefers the native binary when present.
  - *How:* `scripts/build_hook_binary.py`, `claude-plugin/bin/dos-hook{,.ps1}`, `claude-plugin/hooks/hooks.json`.

## `efficiency` — the token-effectiveness verdict (docs/263)

- **`dos efficiency --work W --tokens N [--floor R]`** — classifies `work / tokens` as EFFICIENT / COSTLY / WASTEFUL.
  - *Why:* a run can be PRODUCTIVE yet burn 10x the tokens its work was worth; this relates the work to its price — the question an operator means by "token effectiveness."
  - *How:* `src/dos/efficiency.py` + `src/dos/cli.py`. Non-forgeable by construction — both counts are env-authored (the work git/test-runner witnessed; the tokens the provider billed). WASTEFUL is unit-independent and always-free; COSTLY is opt-in behind a host-armed `floor` (default 0.0 = disabled). Timeless — makes no I/O.

## `verdict_journal` / `observe` — observability as a first-class surface (docs/262)

- **The verdict WAL + `dos observe`** — every verdict is appended to a journal, and `dos observe` reads it back.
  - *Why:* a verdict that is only printed is not observable after the fact; persisting it makes "what did the kernel decide, and when?" a first-class query.
  - *How:* `src/dos/verdict_journal.py`, `src/dos/observe.py`, a new field on `SubstrateConfig` (`src/dos/config.py`); pinned by `tests/test_verdict_journal.py` + `tests/test_observe.py`.

## `noop_streak` / `marker_sensor` — the wait-marker budget runtime lever (docs/259)

- **`noop_streak` generalizes the keep-alive wait-marker budget into a kernel verdict**, wired to a live Stop hook.
  - *Why:* a loop that emits only wait-markers is spinning; the budget caps how long that is tolerated before the run is stopped.
  - *How:* `src/dos/noop_streak.py`, `src/dos/marker_sensor.py`, `src/dos/loop_decide.py`; `scripts/trajectory_audit.py` learns the streak signal. Pinned by `tests/test_noop_streak.py` + `tests/test_marker_sensor.py`.

## `loop_decide` — descendant-progress adopt-wait fix (FQ-509)

- **Stop false-STOPping over a parked-but-committing child** — a parent now adopts a wait when a descendant is still making progress.
  - *Why:* a child parked mid-step but still committing was being read as a stalled subtree and STOPped prematurely.
  - *How:* `src/dos/loop_decide.py`; `tests/test_oracle_and_loop.py` covers the adopt-wait.

## `skills` — `dos-goal-gate` (docs/259 sibling)

- **`dos-goal-gate`** — ground a `/goal` stop-condition on a witness, not self-report, and explain how it composes with the harness `/goal` command.
  - *How:* `src/dos/skills/dos-goal-gate/SKILL.md` (+ the plugin copy); the full-pack skill count rises to 11 (`tests/test_init_skills.py`). Names no host (`tests/test_skill_pack_litmus.py`).

## `benchmark` — two new harnesses (docs/261, docs/260)

- **`witness_ladder`** — sweeps `reward.admit` over the accountability rung to plot value vs witness strength as one monotone curve (the axis no other bench varies).
  - *How:* `benchmark/witness_ladder/`, registered in `benchmark/registry.py`.
- **`loop_authoring`** — proves writing loops beats prompting them, with a prose-arm control.
  - *How:* `benchmark/loop_authoring/`.
