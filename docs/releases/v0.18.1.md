**TL;DR** — One real kernel fix for the phantom lane-lease bug (FQ-532 /
docs/281): a loop that ACQUIREd a lane and then crashed without RELEASE left an
*immortal* lease that the PRE-admission hook enforced against every tool call —
warning on Bash and **hard-DENYing Read/Edit** — until an external SCAVENGE
landed. Both halves are fixed: the live-lease read now self-heals past a
provably-dead orphan, and a contention-only refusal stays advisory so a phantom
lane can never revoke a live session's tools. Patch-safe; genuine cross-process
collision protection is unchanged.

## `lane-lease` — self-heal the phantom orphan (Defect 1, fix)

- **`live_leases` now expires the provably dead.** The structural WAL fold
  (`lane_journal.replay`) returns every un-RELEASEd ACQUIRE as "live" with no
  regard for the clock; a crashed worker's orphan was therefore immortal until a
  separate SCAVENGE writer removed it.
  - *Why:* the PRE-admission hook reads the live set, so an immortal orphan
    enforced a phantom lane (`alpha`/`DTE`/`replan`) on every tool call.
  - *How:* `lane_lease._lease_is_dead` drops a folded lease that is past its own
    `ttl_minutes + grace` (newest of `heartbeat_at`/`acquired_at`, via
    `lease_health.parse_iso`) **or** whose holder PID is confidently dead on this
    host (`proc_delta.probe` — three-valued, foreign-host-blind, never fabricates
    alive). `_expire_dead` is best-effort per-lease (a predicate fault keeps the
    lease).
- **Contention vs reclaim split — the expiry is opt-in at the reader.**
  `live_leases(expire_dead=False)` stays the base structural contract so
  `adopt()` and the orphan-reclaim sweep still **see** the dead orphan to
  transfer/scavenge it; the **contention** consumers pass `expire_dead=True` —
  `acquire()`'s arbitration read and `pretool_sensor.live_leases_for` (the hook
  seam) — so admission self-heals while reclaim still works.
  - *How:* `src/dos/lane_lease.py`, `src/dos/pretool_sensor.py`.

## `admission` — a contention-only refusal stays advisory (Defect 3, fix)

- **A phantom lane can no longer hard-DENY a known-tree tool.** The old
  `reason_class != "" || treeKnown` gate escalated a contention-only refusal (a
  Read/Edit refused solely because some lane was held, with no `reason_class`) to
  a hard `permissionDecision: deny` as soon as the request tree was known.
  - *Why:* a phantom-lane false positive silently revoked Read/Edit while Bash
    only warned — the asymmetry that drove the "route everything through Bash"
    workaround.
  - *How:* `decide.go` now requires a typed `reason_class` (SELF_MODIFY) **or** a
    non-empty known tree that genuinely overlaps; an empty-tree contention refusal
    stays WARN-and-pass regardless of tree-known.

## `hook` — Go twin parity

- **The bundled `dos-hook` fast-path self-heals identically.** `wal.go`'s
  `leaseExpired` + `liveLeasesFromWALAt` port the TTL/heartbeat backstop (the
  hook deliberately does not cross-probe PIDs — the TTL backstop alone closes the
  immortal orphan). The plugin binary is rebuilt for this release so the
  published fast-path carries the fix.

**Fail-safe direction:** a lease the kernel cannot **prove** dead is always kept,
so the expiry can only ever shrink the live set by removing the provably dead —
it never admits a colliding live worker.

**Tests:** `tests/test_lane_lease_expiry.py` (13) + `go/internal/hook/wal_expiry_test.go`
(7) + `decide_test.go::TestReadAgainstContendedLaneWarnsNotDenies`. The full Go
hook package and the Python lease/admission/hook suites are green.
