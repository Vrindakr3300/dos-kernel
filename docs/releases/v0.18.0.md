## `cooldown` — deterministic tie-break (fix)

- **`cooldown_verdict` no longer leaks input order into the verdict.** When two
  attempts shared an `attempted_at_ms` but carried different outcomes (DRAINED's
  6h window vs ERROR's 30m), a stable time-only sort made `mine[-1]` — and thus
  the verdict — depend on the order the host gathered the records: CLEAR one way,
  RECENTLY_ATTEMPTED the other.
  - *Why:* a verdict that flips under `reversed(history)` is non-deterministic; an
    operator re-gathering the same attempts could get a different pick decision.
  - *How:* at a timestamp tie, select the tied-latest attempt with the LONGEST
    wall (`policy.window_for`, then `ts` as a total order). Order-free *and*
    preserves the module's documented "fail toward cooldown" bias. `src/dos/cooldown.py`.
  - Caught by `tests/test_prop_cooldown.py::test_verdict_is_invariant_under_history_order`
    (docs/273); all 30 cooldown tests green.

## `export` — the resumable forward drain (docs/266 Phase 4)

- **`--since auto` + a bounded `--follow` loop shipped and test-pinned.** The
  exporter now reads a persisted high-water offset, ships past the last shipped
  `seq`, and writes the new mark back to `.dos/export-cursor.<transport>` (a
  per-transport suffix so file/otlp drains track independent progress).
  - *Why:* a fleet wants to drain the verdict journal incrementally without
    re-shipping what already left.
  - *How:* `src/dos/export_cursor.py` (a WAL-adjacent data module, fail-soft on
    read → 0 and write → False); `--follow` is a BOUNDED poll→drain→persist→sleep
    that terminates on `--follow-max`/^C, never a `while True` daemon (the
    `dos notify` posture). Persist fires ONLY when a real transport actually
    shipped — null/dry-run/failed drains hold the cursor.
  - Pinned by the Phase-4 cases in `tests/test_export_cli.py` (19 passed).
