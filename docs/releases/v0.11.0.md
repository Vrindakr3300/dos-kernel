**TL;DR** — A feature minor on top of v0.10.0. Two new operator-facing syscall
surfaces (`dos status <run_id>` and `dos arg-provenance`), a pluggable
loop-STOP seam (`stop_policy`) that lets a host opt a decision class into a real
halt without touching the kernel, the completion verdict that closes a loop on a
real DONE, and the `churn` fold that collapses the no-op-archive commit flood.
Anyone running a dispatch loop against DOS should care about `status`,
`completion`, and `churn`.

## `dos status <run_id>` — the folded fact (docs/120 Phase 2/3)

- **What changed** — a new read-only verb + MCP tool that folds every kernel
  state surface for one run into a single digest.
  - *Why:* the operator (and an A2A dashboard) wants one "where is run X" answer,
    not five separate reads of the journal/ledger/verdict surfaces.
  - *How:* `src/dos/status.py` is the pure `StatusDigest` fold (Phase 1);
    `src/dos/cli.py` `dos status` + `dos_mcp` `dos_status` are the boundaries.
    The no-`claimed`-key invariant is pinned at BOTH the CLI and MCP boundary.
    The `--stopped` flag resolves to an automatic `suspended OR STALLED`
    predicate.

## `stop_policy` — a pluggable loop-STOP seam

- **What changed** — a `StopPolicy` Protocol + `StopVerdict` so a host can OPT a
  decision class into a real loop STOP, where the kernel default only ever
  SURFACES a blocker.
  - *Why:* some decision classes (a resource floor breach, a human-gate) should
    actually halt the loop, but that policy is the host's to declare, not the
    kernel's to hardcode.
  - *How:* `src/dos/stop_policy.py` — fail-to-DEFER (a raising/bad policy
    degrades to "do not stop"), resource-floor AND. The default is unchanged;
    `dos decision-class` is the driver surface.

## `arg-provenance` — distrust the declared FK args (docs/143)

- **What changed** — the `ProvenanceVerdict` rung that grades whether a
  tool-call's declared foreign-key arguments actually trace, plus the
  `dos arg-provenance` operator verdict.
  - *Why:* a model self-reports the ids it acted on; the substrate should not
    take the declared extent on faith.
  - *How:* `src/dos/arg_provenance.py` + `src/dos/cli.py`. Real-data hardened on
    a live `gemini-3-flash` run: full ISO-8601 timestamps, UUIDs/32-hex demanded
    whole (never split on `-`), and email-local digit discriminators are no
    longer mis-flagged as FKs.

## `churn` — collapse the no-op-archive commit flood

- **What changed** — `churn.decide_coalesce(ChurnState) -> CoalesceVerdict`: when
  a no-op dispatch archive repeats the same cause as the prior commit, amend the
  prior commit instead of adding a new one.
  - *Why:* the push gate kept peers clean but the LOCAL commit flood remained —
    measured 199 commits → 24 picks, one phrase recurring 22× in the window.
  - *How:* `src/dos/churn.py`, layered on `event_severity` (only a NOOP
    coalesces), keyed on the host's opaque `cause_key`. Pure kernel, I/O on the
    edge; the full per-run audit still lives in the README tree.

## `completion` — the live completion verdict (docs/117 Phases 1-3)

- **What changed** — a real completion verdict + a stop-on-done gate inside
  `decide()`, plus the `scope_source` rung that distrusts a declared extent.
  - *Why:* a loop should close on a verified DONE, not on the agent narrating
    "I'm finished."
  - *How:* `src/dos/completion.py`, `src/dos/loop_decide.py` (stop-on-done),
    `src/dos/scope_source.py` (the ScopeSource rung, Phase 4).
