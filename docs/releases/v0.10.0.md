**TL;DR** ‚Äî This release centers on `dos.event_severity`, the pure kernel behind
the dispatch-family noise filter the `job` host consumes: it turns each
bookkeeping event's already-computed facts into one typed `Severity`, and (new in
this release) renders the operator-facing commit-subject headline mechanically so
the run-ordinal leak (`185th /replan`) becomes structurally impossible. It also
batches the accumulated kernel work landed since the `v0.9.0` tag-only release.

## event_severity ‚Äî the dispatch-family noise-filter kernel

- **`classify_event(EventState) -> Severity`** ‚Äî a pure function turning an event's
  facts (the `verdict=<X>` token, pick count, first-occurrence) into one typed level:
  `SHIPPED` > `BLOCKED-NEW` > `NOTICE` > `NOOP`.
  - *Why:* the dispatch family committed + pushed an archive every iteration ‚Äî even
    0-pick drains and repeated rate-limits. Severity = `(verdict, first_occurrence)`
    so a FIRST-seen blocker is actionable (`BLOCKED-NEW`) while the *same* blocker
    recurring collapses to `NOOP` ‚Äî that predicate is what kills the repeated-blocker
    flood.
  - *How:* `src/dos/event_severity.py`; reuses `tokens.normalize_token` (folds legacy
    `WEDGE -> BLOCKED`) so the classifier never re-lists the verdict vocabulary.
- **`subject_lead_token(EventState) -> str`** (new this release) ‚Äî the mechanical
  commit-subject headline (`quiet sweep` / `inbox promoted: N` / `N picks shipped` /
  `drained` / `blocked (‚Ä¶)`), derived from the same `EventState` `classify_event`
  reads so the headline and the gate verdict can never disagree.
  - *Why:* the host's Phase-1 *prose* rule ("the ordinal NEVER appears") drifted back
    to `185th /replan` ‚Äî a model retypes the subject every run. A function cannot.
  - *How:* the run ordinal is not a field of `EventState`, so a caller building the
    subject from this token cannot leak it. PURE ‚Äî no I/O. +20 replay tests.
- **`sink_threshold` / `admits`** ‚Äî the one I/O concession (a single `os.environ.get`),
  kept beside the classifier so a consumer has one import; per-sink `JOB_DISPATCH_*`
  thresholds make the common case quiet (only `SHIPPED` + first-seen blockers clear
  the default push bar).

## Kernel work batched since v0.9.0

- **`provider_limit`** ‚Äî canonical `{TRANSIENT_OVERLOAD|USAGE_WINDOW|HARD_QUOTA|NONE}`
  category + policy + mappers (PLC1) so the three legacy taxonomies collapse to one
  output vocabulary.
- **`lane_journal` / `lane_lease`** ‚Äî supervised-child adoption (the ADOPT transfer op)
  and the WAL-authoritative arbiter read-flip.
- **`id_alloc`** ‚Äî atomic monotonic id allocator (the TAG_COLLISION structural fix) +
  win32 `EACCES` retry on the contended `O_EXCL` create.
- **`arbiter`** ‚Äî refuse `UNKNOWN_LANE` instead of silently auto-picking a substitute;
  prefer a sibling-disjoint lane on bare auto-pick.
- **`overlap_policy` / `lane_overlap`** ‚Äî the disjointness scorer (Axis 7) opened with
  an honest docstring on the known-hazard elbow.
