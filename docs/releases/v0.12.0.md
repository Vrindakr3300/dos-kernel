**TL;DR** ‚Äî DOS spent its history hardening the *verdict*; the live EnterpriseOps-Gym run proved
that is necessary but not sufficient ‚Äî a *sound* verdict, enforced too hard, lost points. This
release adds the missing axis: a closed, typed **intervention ladder** that makes acting on a
verdict cost less than the harm it prevents, and a new **loop-economics stall reader**
(`tool_stream`) ‚Äî the first DOS lever on this benchmark that moves success *up* and whose value is
*independent of minting* (it fires on any looping model, including a strong one). Both leaves are
pure, unit-tested, and *run* in faithful simulators.

## `intervention` ‚Äî harden the actuation, not just the verdict (docs/144)

- **The closed actuation ladder** `OBSERVE < WARN < BLOCK < DEFER` ‚Äî the dual of `dos.reasons`.
  - *Why:* the live run showed a SKIP nudge cost ‚àí9 pp on a true-positive catch; WARN-only
    recovered it. Detector-soundness and intervention-safety are orthogonal properties.
  - *How:* `src/dos/intervention.py` ‚Äî `Intervention` enum, `InterventionLadder` (closed +
    `extend`-not-mutate), the measured cost order (BLOCK *below* DEFER ‚Äî it preserves the turn).
- **Confidence-gated escalation + the turn-preserving BLOCK** ‚Äî `choose_intervention` /
  `assess_confidence` map a verdict's confidence to a rung under a refuse-LESS-only policy
  (validated twice); `synthetic_corrective_result` returns a corrective tool-result in place of
  the minted call, so the agent gets a correction without losing the turn (`dos_blocked` excluded
  from the corpus to prevent re-poisoning).
- **`intervention_eval` + `dos intervention-eval`** ‚Äî score a policy by NET TASK DELTA (not
  verdict accuracy); the dangerous-cell `wasted_disruption_rate` is the ‚àí9 pp made measurable.

## `tool_stream` ‚Äî the loop-economics stall reader (docs/145)

- **A new pure kernel leaf** `classify_stream(ToolStream, StreamPolicy) -> StreamVerdict`
  (`ADVANCING` / `REPEATING` / `STALLED`) ‚Äî `liveness.classify`'s lateral sibling, re-aimed off
  git onto the in-process tool-result stream; reuses `churn`'s consecutive-identical run-length.
  - *Why:* a cheap model thrashes (re-reads, polls, loops to timeout half-done); the paper shows
    success decays monotonically with horizon. That slice is *not* planning ‚Äî the agent had the
    value and failed to use it.
  - *How:* byte-clean by the ¬ß5a test ‚Äî the agent did not author the *identity* of its own
    repeated env-results, so REPEATING is provenance-of-repeated-output, never a satisfaction
    predicate. On REPEATING a consumer re-surfaces the value the agent already holds (a WARN,
    never a cut ‚Äî eventual-consistency polling is a legitimate repeat, exempted via `ignore_tools`).
- **`tool_stream_eval` + `dos tool-stream-eval`** ‚Äî score a stall policy by net recovery
  (recovered-rate headline, false-resurface-rate dangerous cell); exit 0 iff net-positive.
- **The `[tool_stream]` config seam** ‚Äî `SubstrateConfig.stream_policy` (`repeat_n` / `stall_n` /
  `ignore_tools`, OVERRIDE-on-present) + a `stall reader` row in `dos doctor`.

## `arg_provenance` ‚Äî real-data hardening (docs/143)

- Bare-integer-FK recall + a quantity-name stoplist, lifting live recall 55 %‚Üí83 % while holding
  precision at 0 % false-nudge on 249 real resolved calls (the `gemini-3-flash` run).

## The simulators that run the leaves ‚Äî plumbing + emergence, NOT magnitude (docs/148 ¬ßL1)

> **‚öÝ Honesty note (corrected post-tag).** These simulators prove the mechanism *fires* and that
> the delta is *emergent* (‚Üí 0 when the failure they target ‚Üí 0). They do **not** prove the
> magnitude ‚Äî that is governed by an *assumed* failure-rate the sim cannot measure. The precedent
> is exact: the arg_provenance sim said **+11.3 pp**, but the **real `gemini-3-flash` run measured
> ~0** (a strong model doesn't mint) and **~+1 pp verifier-pass** on a deliberately-weakened arm
> (`RESULTS.md`). Expect the same order-of-magnitude collapse for the stall reader. **The only
> trustworthy number is the real gym run, which is not yet done for `tool_stream`.**

- **`simulator.py`** (arg_provenance): Integrity slice +9.7 pp *simulated*, feasible-rate flat, 0
  false-nudges ‚Äî emergent (‚Üí 0 as the mint-rate ‚Üí 0). **Real run: ~0 / ~+1 pp** (`RESULTS.md`).
- **`stall_sim.py`** (tool_stream, new): the headline scales with the *assumed* loop-rate
  `p_stuck`, which has **no real-data anchor** ‚Äî at the likely-real strong-model rate it is **~+3 pp
  simulated**, and the `--honest` sweep makes that dependence explicit (run
  `python -m benchmark.enterpriseops.stall_sim --honest`). **Not yet measured on the real gym.**
  The `--ignore-pollers` knob drops false-resurfaces to 0 at zero cost to recovery (the one
  property that *is* mechanism-true, not magnitude).
