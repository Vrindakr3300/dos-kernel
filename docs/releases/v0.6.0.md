**TL;DR** — v0.6.0 widens the operator surface with three read-only/driver additions
that all rest on the same trust seam: `dos plan` shows the *shape of the work* (claimed
vs adjudicated), `dos lease-lane` lets an external orchestrator keep DOS's
collision-*prevention* guarantee, and `dos loop` is the fleet's init process. Underneath
sits a tier of concurrency-safety fixes (the audit findings) so the lock/lease plane is
race-clean on Windows and across processes. The full kernel suite is green (815 passed).

## `dos plan` — the work-terrain board (the third TUI projection)

- **What changed** — a new read-only projection beside `dos top` (what's running) and
  `dos decisions` (what needs me): `dos plan` answers *what's the shape of the work and
  how far has it shipped*.
  - *Why:* a self-reporting plan view is a self-narrating worker; the kernel is built to
    distrust exactly that.
  - *How:* it is a `verify()`-fan-out, not a plan reader — a declared `plan_source` seam
    (`src/dos/plan_source.py`) yields candidate `(plan, phase)` rows and
    `oracle.is_shipped` rules on each. The headline cell is the **divergence flag** (plan
    says SHIPPED, oracle says not). `src/dos/plan_board.py` + the `[tui]` rich skin
    `plan_board_tui.py`; `dos plan [--once|--json|--source NAME] [PLAN PHASE …]`.
- **Conservative by design** — the generic `MarkdownPlanSource` under-harvests a foreign
  plan dialect (→ "(no plans)" + git floor) rather than mine prose for phantom phases.
  Other dialects ship a `dos.plan_sources` plugin.

## `dos lease-lane` — the orchestrator is a driver (docs/98)

- **What changed** — a durable lane-lease write-back verb over the pure arbiter, plus the
  FleetHorizon orchestrator arm that benchmarks it.
  - *Why:* the one property riding on DOS owning the dispatch loop is collision-
    *prevention*. A cross-process harness `parallel()` has no shared lease list, so two
    branches silently admit a colliding tree — collision *detected*, not *prevented*.
  - *How:* `dos lease-lane acquire/release/live` (verdict = exit code) WALs the
    arbitrated lease to the lane journal, so a sibling process reconstructs `live_leases`.
    `benchmark/fleet_horizon/orchestrator.py` (the `LeaseBook` seam) + `harness_loop.py`
    (the harness arm) + `--orchestrator-sweep`.
- **Measured** — seed 1729, 8×30, shared_ratio=0.3: DOS-native and disciplined-harness
  (with write-back) both reach prevention=100%; naive harness prevention=0% (10 detected,
  10 silent). Honesty invariants hold across all arms — the trust axis is orthogonal.

## `dos loop` — the fleet supervisor (docs/99)

- **What changed** — the init/PID-1 of a DOS fleet: a pure `supervise()` verdict + a
  `drivers/supervisor.py` driver + the `dos-supervise-loop` shipped skill.
  - *How:* `src/dos/supervise.py` (the pure verdict), `src/dos/cli.py` `dos loop`, lane-
    journal support for the supervisor's bookkeeping.

## Kernel hardening — the concurrency-safety tier

- **Value-keyed CAS lock-steal** — `_filelock` factors a shared compare-and-swap steal so
  `archive_lock` + the home lock + the lane-lease `_Mutex` all close their TOCTOU /
  double-admit windows (the CRITICAL audit finding).
- **Case-folded path algebra** — `_tree`/`scope`/`arbiter`/`stamp` casefold path and
  lane-name comparisons, killing a Windows false-admit + a `SELF_MODIFY`-guard bypass.
- **Birth-skip liveness** — `journal_delta`/`dispatch_top` exclude a lease's own BIRTH
  acquire from the ADVANCING event count, so a freshly-acquired lane can read SPINNING.
- **Oracle gates ON by default** — the FQ-390 plan-id-collision gate and the #326 soak
  suppression are now wired into `oracle.is_shipped` on the verify surfaces.
- **Pre-dispatch health gate** + `packet_sidecar` producer-side shippability verify
  (FQ-419/FQ-420) + `timeline` per-stage duration column + `stamp.parse_phase_labels`.
