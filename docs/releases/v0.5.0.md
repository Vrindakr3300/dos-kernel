**TL;DR** — v0.5.0 widens what DOS can *verify* past the git fossil (a CI/Checks
oracle for the non-git surface) and lands the typed-verdict + `scope` surface,
a copy-me reference driver with a generic `--driver` loader, a round of kernel
hardening, and the full public-repo adoption surface. Anyone adopting DOS as a
referee — or publishing the `dos-kernel` dist — is the audience.

## The CI/Checks oracle — verify() past the git fossil (docs/85 move-B)

- **A non-git artifact oracle** — `verify()` reads the git fossil (existence +
  ancestry + subject grammar); the CI oracle answers the signal git is blind to:
  *is the build green at this commit?*
  - *Why:* docs/85 §2 names three ways to extend the verifiable surface; only one
    is a kernel change. This is the canonical **move (B)** — a new driver, not a
    kernel edit.
  - *How:* `src/dos/drivers/ci_status.py` (+ `tests/test_ci_status.py`), reading
    the workflow shipped at `.github/workflows/ci.yml`.

## Typed-verdict surface + the scope verb (docs/86/87)

- **A structured verdict contract** with its own modular CLI dispatcher — the
  verdict is a typed envelope, registered and dispatched by name.
  - *How:* `src/dos/scope.py`, `src/dos/verdict.py`, `src/dos/verdicts.py`,
    `src/dos/verdict_cli.py` (+ `tests/test_scope.py`,
    `tests/test_verdict_contract.py`, `tests/test_verdicts.py`,
    `tests/test_verdict_cli.py`).

## A copy-me reference driver + the generic loader

- **`workshop` reference driver** — a self-contained host-policy template (unlike
  `job.py`, it declares its taxonomy inline): two concurrent tree-disjoint cluster
  lanes, an exclusive release lane, `--lane` aliases, and the docs-prefix
  concurrency trick, all in one file.
  - *How:* `src/dos/drivers/workshop.py` (+ `tests/test_drivers_workshop.py`).
- **`--driver <name>` resolves by convention** — `dos.drivers.<name>.<name>_config`,
  no hardcoded host name; `--job` becomes the back-compat alias for `--driver job`.
  A dotted / path-traversal name is rejected up front (closes a traversal surface).
  - *How:* `_resolve_driver_config` in `src/dos/cli.py`.
- **`dos init` scaffolds a usable `[lanes]` table** — generated from the repo's
  top-level layout (capped, noise-dirs skipped) instead of a dead whole-repo lane.
  - *How:* `_render_init_config` / `_INIT_LANE_MAX` in `src/dos/cli.py`.

## Kernel hardening

- **Workspace-aware admission predicates** — `built_in_predicates(workspace=, config=)`
  so the SELF_MODIFY guard reads cached `WorkspaceFacts` instead of re-probing disk.
  - *How:* `src/dos/admission.py`, `src/dos/arbiter.py`, `src/dos/self_modify.py`,
    `src/dos/_tree.py`, `src/dos/lane_overlap.py` (+ `tests/test_lane_overlap.py`).
- **The stamp filepath rung** — the ship-stamp grammar gains a filepath signal.
  - *How:* `src/dos/stamp.py` (+ `tests/test_stamp_filepath_rung.py`).
- **Preflight sidecar status** — the prompt-sidecar loader reports `absent` vs
  `corrupt` instead of collapsing both into a bare markdown fallback (FQ-420).
  - *How:* `src/dos/preflight.py` (+ `tests/test_preflight_sidecar.py`).

## Public-repo adoption surface & docs

- **Adoption scaffolding** — README/SECURITY rewrites, `.github/` issue +
  PR templates and CI workflow, `examples/demo/` (a runnable `verify` demo), and
  PyPI packaging metadata (readme, keywords, classifiers) on `pyproject.toml`.
- **Trimmed to the kernel + how to use/extend it** ahead of the public cut —
  host-specific planning narrative and strategy docs were moved out so the repo
  reads as a trust substrate, not a doc archive.
- **Open-research-areas series (docs/88–95)** — the adjudicator population, the
  lane-as-region-lock framing, value-aware picker + predicate-precision plan notes,
  live non-git verification, checkpoints/recovery, and the OS-level evidence rung.
- **Trajectory-audit dev tooling** — `scripts/trajectory_audit.py` (+ the skill and
  `tests/test_trajectory_audit.py`): fuse what an agent *said* against what the
  kernel *adjudicated*.
