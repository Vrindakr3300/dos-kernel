**TL;DR** ‚Äî This release adds three operator-facing surfaces to the kernel (`dos attest`/`dos verify-receipt`, the `pick_priority` freshness picker, and new `lease_health`/`tokens` verdicts), bundles DOS into a single Claude Code plugin, and finishes reaping the last host-specific literals out of the kernel. Alongside the code, the fleet-scale payoff research program (docs/245 F1‚ÄìF4) and the arXiv paper landed. No breaking ABI change ‚Äî `verify`/`arbitrate`/`refuse` signatures and the verdict vocabularies are unchanged, only added to.

> **Version note:** the working-tree version markers had lagged at `0.13.0` while `v0.14.0` was already tagged and shipped; this cut bumps both markers straight to `0.15.0`, closing the lag. The previous release (`v0.14.0`) was the picker substrate + fold-site witnesses + commit-audit + the supervise seam.

## `cli` ‚Äî three new operator verbs (docs/246, docs/254)

- **`dos attest` + `dos verify-receipt`** ‚Äî mint a portable, signed receipt for a verdict and verify it later from anywhere.
  - *Why:* a verdict is only useful downstream if a consumer who didn't run it can trust it; the receipt is the carryable proof.
  - *How:* `src/dos/cli.py` (docs/246 Phase 1) + `src/dos/attest.py`.
- **`dos pick_priority`** ‚Äî surface the freshness-ordered pick queue from the CLI.
  - *Why:* a fleet that always re-picks the same lane starves new work; freshness is a within-tier tie-breaker (never-attempted ‚Üí least-recently-attempted).
  - *How:* `src/dos/pick_priority.py` + `src/dos/cli.py` (docs/254). The ordering is a refuse-more-only floor ‚Äî it reorders candidates, never admits one the gate would refuse.
- **`dos` quickstart** ‚Äî reach the workshop reference driver in one unshadowed command.
  - *How:* `src/dos/cli.py`, `claude-plugin/skills/dos-setup/SKILL.md`, `scripts/build_plugin.py`.

## `decisions` / verdicts ‚Äî new failure classes (FQ-520)

- **`lease_health` CHILD_CHURNING** ‚Äî a live child whose picks have already shipped.
  - *Why:* a child that keeps holding a lease after its work is done is churning, not progressing ‚Äî a distinct verdict from a stalled or parked child.
  - *How:* `src/dos/lease_health.py`.
- **`tokens` CHILD_PARKED_PRE_STEP9_NO_ENVELOPE** ‚Äî a new `BlockedReason` for a child parked before emitting its envelope.
  - *How:* `src/dos/tokens.py`.

## `loop_decide` ‚Äî the breaker migration (docs/258)

- **The inline retry counters move onto `dos.breaker`** plus a new `REPLAN_STALLED` rung.
  - *Why:* failure-counting was duplicated ad hoc inside the loop; the kernel already has a pure circuit-breaker primitive, so the loop should ride it.
  - *How:* `src/dos/loop_decide.py` now delegates to `dos.breaker`; `tests/test_oracle_and_loop.py` covers the rung.

## `oracle` / kernel ‚Äî the host-reaping (genericization) pass

- **The last host-specific literals leave the kernel** ‚Äî the `orchestration` lane globs, the vendor `claude` binary, the phased-plan literals on the `verify()` demotion path, and the host dispatcher prefixes are now data, not hardcoded.
  - *Why:* the kernel must name no host and no vendor; these were the residue a userland-reap audit found.
  - *How:* `src/dos/oracle.py`, `src/dos/arbiter.py`, `src/dos/_job_policy.py`, `src/dos/sibling_scan.py`, `src/dos/claim_ttl.py`, `src/dos/event_severity.py`, `src/dos/supervise.py`, `src/dos/cli.py` (audit: `docs/_audits/USERLAND_REAP_AUDIT_2026-06-08.md`).
  - *Impact:* behavior is unchanged for the generic default; host policy that relied on the old literals now declares them in `dos.toml`/a driver.

## `drivers` / distribution ‚Äî one-install Claude Code plugin

- **The Claude Code plugin bundles the DOS hooks, MCP server, and skill pack into one install.**
  - *Why:* the lowest-friction adoption surface ‚Äî a stranger installs one plugin and gets the hooks, the `dos_mcp` tools, and the generic skills together.
  - *How:* `claude-plugin/` (manifest, `.mcp.json`, hooks, 11 skills) + `scripts/build_plugin.py`; pinned by `tests/test_plugin_manifest.py`. A throwaway-repo smoke (`examples/demo/`) proves the plugin works isolated.
