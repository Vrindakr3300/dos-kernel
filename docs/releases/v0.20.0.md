**TL;DR** — Two landings since v0.19.0. A real performance fix collapses the ship
oracle's file-path backstop from one `git log` subprocess *per named file per pair*
into a single windowed `--name-only` scan shared across the whole batch — measured
~20x faster, with verdicts proven byte-identical to the per-pair path. And a new
`observe` surface lets `dos top` answer the question an operator actually asks:
"how many things did DOS help with?" Anyone running the oracle over a large pick
set (the `/dispatch-loop` acquire/pre-launch/orphan-sweep path) gets the speedup
for free; the help summary is additive.

## `oracle` — batch the file-path backstop's git log (perf, docs/284)

- **The AAR-FQ230 file-path backstop now does one `git log` scan per batch, not one
  per named file per pair.** The per-pair overlap test became a pure in-memory
  lookup against a shared `{path: [(sha, subject), …]}` cache.
  - *Why:* the backstop shelled `git log --oneline -800 -- <file>` once per named
    file — 364 git subprocesses for a 262-pair job snapshot, ~19s, the single
    biggest per-iteration deterministic tax in a `/dispatch-loop`. Every call asked
    the same git history the same question; only the per-pair overlap differed.
  - *How:* `phase_shipped.build_batch_filepath_cache` harvests the union of every
    pick's files and drives `_build_filepath_log_cache` (one
    `git log --name-only --no-merges` over the union); `oracle._grep_batch_in_process`
    and `phase_shipped.main()`'s `--batch`/`--check-packet` build it once before the
    per-pick loop. Measured 20.9x at 30 files (617ms → 29ms).
- **Byte-identical by construction — the never-under-count pin.** A buggy batch path
  would drop a genuinely-live pick, so parity is the load-bearing invariant.
  - *How:* a saturated union window returns `None` and the caller re-runs the exact
    per-file path; `%h` (not `%H`) keeps the `sha`/`summary` fields equal;
    `--no-merges` on **both** paths removes git's per-path merge-simplification
    ambiguity at the source (a merge is never a phase's ship of record). Verified
    129/129 `src/dos` files byte-identical, pinned by
    `tests/test_filepath_backstop_batch.py`.

## `observe` — surface "DOS helped with N things" (feat, docs/285)

- **A new operator-help summary reads the verdict trail and reports how many things
  DOS materially helped with** — the last rung of the observability ladder, from
  raw verdicts up to a single operator-facing count.
  - *Why:* the kernel emits per-event verdicts, but an operator wants the rollup:
    "was this worth running?" The summary answers it from evidence DOS already
    recorded, not a self-report.
  - *How:* `src/dos/help_summary.py` + the `dos` CLI surface (`src/dos/cli.py`),
    pinned by `tests/test_help_summary.py`.

## `tooling` — release-bump sweeps the FTUE version literals (build)

- **`scripts/release_bump.py` now rewrites the onboarding doc banners and skill-pack
  version samples** in lock-step with the package version.
  - *Why:* the v0.19.0 cut left the FTUE docs at 0.18.0 — a drift hidden by stale
    editable-install metadata. The sweep makes the banner literals part of the bump.
  - *How:* the `docs` bump target keys on the old→new pair and rewrites only the
    literals naming the old version; `build_plugin.py` resyncs the bundled skill copies.
