**TL;DR** — DOS gains its last hackability seam (pluggable admission predicates,
with a built-in self-modification guard) and its lowest-friction adoption surface
(an MCP server that speaks the syscalls to any agent host with zero Python
coupling). The generic skill pack is fully de-hardcoded, and a real fleet
benchmark now exercises the kernel end-to-end. First tagged release.

## Admission predicates — the safety axis of hackability (ADM, docs/73)

- **The arbiter's admission rule is now pluggable.** An admission predicate is a
  pure `(request, live_lease, config) -> AdmissionVerdict` resolved from the
  `dos.predicates` entry-point group; the arbiter runs the built-ins *plus* every
  registered predicate, conjunctively.
  - *Why:* a workspace can add "refuse over the monthly token budget" or "refuse
    outside business hours" without forking the arbiter.
  - *How:* `src/dos/admission.py` (`AdmissionVerdict`, `run_predicates`,
    entry-point discovery, the behavior-preserving `DisjointnessPredicate`).
- **The one invariant that keeps an open predicate set safe: predicates may only
  REFUSE, never force-admit.** `AdmissionVerdict` has only `.admit()`/`.refuse()`
  — no "admit harder" — so a buggy or hostile predicate is *structurally*
  incapable of loosening admission. A predicate that raises is fail-closed-refused.
  `--force` stays the sole override.
- **Built-in `SELF_MODIFY` guard.** Refuses a lease whose tree intersects the
  orchestrator's own running code (the arbiter, classifiers, reason vocabulary,
  config seam — `dos.self_modify._DISPATCH_RUNTIME_FILES`). A live loop must not
  rewrite the kernel that is adjudicating it.
  - *How:* `src/dos/self_modify.py`; `SELF_MODIFY` added to `reasons.BASE_REASONS`
    (category `MISROUTE`) so the refusal is emittable/verifiable/refusable/`dos
    man`-documented.
- **`dos doctor` lists the active admission predicates** (built-ins then any
  discovered plugin), so an operator can see exactly what gates a lease.

## MCP server — DOS as a tool any agent can call (docs/80)

- **`pip install dos[mcp]` ships a FastMCP server (`dos-mcp`)** exposing
  `verify` / `arbitrate` / the structured-refusal vocabulary / `doctor` as MCP
  tools — so Claude Desktop, Cursor, Cline, or an Agent-SDK app can call the
  referee with no Python coupling (JSON over stdio).
  - *Why:* the lowest-friction adoption surface for the kernel.
  - *How:* `src/dos_mcp/` — a separate top-level package that *consumes* `dos`;
    nothing under `src/dos/` imports it. The `mcp` dependency lives only in the
    `[mcp]` extra; the kernel's own deps stay PyYAML-only.
- **Each tool builds a `SubstrateConfig` per call and passes it explicitly** into
  the kernel, rather than mutating process-global active state — correct for a
  long-lived server fielding concurrent workspaces.
- **Shared workspace readback.** The four-table `dos.toml` readback is now one
  implementation (`config.load_workspace_config`) used by both the CLI and the
  server, so they can't drift.

## Skill pack genericization (SKP, docs/74)

- **The five reference skills (next-up / dispatch / replan + the two loops) read
  every host-specific from `dos doctor --json` / `dos.toml`** — ship grammar via
  `[stamp]`, the archive dir via the new `paths.runs`, dispositions under
  `<paths.next_packets>` — instead of any hardcoded host path or lane.
  - *Why:* the shipped baseline must name no host (the "skill imports no host"
    litmus); a grep of a shipped skill for a host dir / job lane / job commit
    prefix returns nothing.

## FleetHorizon benchmark (docs/81)

- **An open-loop-vs-closed-loop A/B that drives the *real* kernel** (`is_shipped`,
  `arbitrate`, `run_id`, `lane_journal`) against *real* git repos — a fleet of N
  concurrent long-horizon efforts on shared state, the horizon×fanout axis
  single-PR benchmarks structurally cannot see.
  - *How:* `benchmark/fleet_horizon/` (integrity axis today; the velocity /
    verified-per-$ axis is specified in docs/81).
