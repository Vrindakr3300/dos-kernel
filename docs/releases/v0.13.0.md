**TL;DR** — This release grows the kernel's detector surface and hardens its evidence
floor. Two new pure detectors (`dangling_intent`, `precursor_gate`) join the byte-clean
family, `evidence.derived_witness` closes the agent-selected-arithmetic laundering hole,
and `tokens` gains a drain-vs-wedge `BlockedReason`. Alongside the kernel work, DOS earns
its first **third-party-scored** result: a $0 offline replay over the Toolathlon-Trajectories
corpus, plus a third in-trace detector (`terminal_error`) — a high-precision, additive slice
that reaches frontier failures the other detectors miss. If you consume the evidence seam
or the EOG/EnterpriseOps detector axis, this is the release you want.

## `evidence` — `derived_witness`: the byte-inequality floor, lifted to a derivation (docs/156)

- **What changed** — `derived_witness` mints a non-forgeable belief for a *derived* value, not just a directly-attested one.
  - *Why:* an agent that hand-picks the arithmetic over otherwise-trustworthy inputs can launder a forged number through a real-looking computation. The old floor only graded directly-attested bytes, so a derivation slipped under it.
  - *How:* `src/dos/evidence.py` — the result rides the non-forgeable rung **iff** all operands are non-forgeable, the operation is *declared* (not agent-selected), and the stated result re-derives from the operands. It is byte-inequality (docs/141) moved one level up: base case = attested operands, inductive case = a declared op over them.

## `dangling_intent` — a new pure detector for premature completion (docs/150)

- **What changed** — a new kernel module that fires when the agent's terminal narration claims an intent it never carried out.
  - *Why:* "premature completion" (the agent stops believing it finished) is a real failure slice. A byte-clean sub-slice of it is detectable without a satisfaction predicate.
  - *How:* `src/dos/dangling_intent.py` — a pure detector over the trajectory; advisory, WARN-shaped (the live EOG A/B taught that a correct detector can still be net-harmful if enforcement is disruptive, so it reports rather than blocks). Proven on the EnterpriseOps-Gym corpus.

## `precursor_gate` — did the mandated precursor fire before this mutation? (docs/147)

- **What changed** — a new kernel gate (+ its eval harness) that checks a required precursor tool ran *before* a guarded mutation, plus a `dos precursor-gate` CLI surface.
  - *Why:* a fourth slice of the trust problem — not "did it ship?" but "did the obligatory step happen first?" A mutation with no preceding precursor is a byte-clean tell.
  - *How:* `src/dos/precursor_gate.py` (the gate) + `src/dos/precursor_gate_eval.py` (the per-axis eval) + `src/dos/cli.py` wiring + a `[precursor]`-style grammar in `config.py`. A `tool_name`-membership scan (not a full trace), REFUTED→WARN, WARN-only by output type — the same advisory discipline as the other detectors.

## `tokens` — a drain-not-wedge blocked reason

- **What changed** — a new `lane_all_inflight_or_deferred` `BlockedReason`.
  - *Why:* a lane whose work is all in-flight or deferred is *draining*, not *wedged* — conflating the two mislabels healthy backpressure as a stall.
  - *How:* `src/dos/tokens.py` — the new reason lets the loop distinguish "nothing to do right now, by design" from "stuck."

## `benchmark` — the third-party-scored replay + a third detector (docs/157, docs/158)

- **What changed** — `benchmark/toolathlon/`: a $0 offline replay that runs the shipped detectors over a third-party-labeled corpus, and a new third in-trace detector, `terminal_error`.
  - *Why:* every prior result was DOS scoring DOS. Toolathlon-Trajectories is labeled by someone else — the first time the purchase is graded independently. The honest finding: the detectors fire at high precision but ~2–3% recall, and purchase *vanishes* on the strongest models (DETECT-not-FIX; the EOG ceiling reproduced on a public benchmark).
  - *How:* `benchmark/toolathlon/` (replay + dataset + trajectory + viz + durable `replay_all_rows.csv`, 7,116 records over 22 models). `terminal_error` (`trajectory.py`) reads the env's terminal **error envelope** (MCP-gateway-authored, not agent-forgeable) with a *tight* structured-envelope grammar — **95.5% precise, +1.5pp recall** (union pair→trio 4.72%→6.24%, +32% relative), **83 of 84 catches net-new** (missed by `dangling_intent`+`tool_stream`), including frontier-model failures `tool_stream` doesn't catch. Its value is precision + additivity + byte-cleanliness, not magnitude. The tight grammar is load-bearing: the rejected *loose* substring grammar (`error`/`failed`/`not found`/4xx-5xx) measures 25.6% false-alarm — unusable — vindicating the structured-envelope choice. (These are the live, CSV-reproduced figures; the docs/158 design doc carries an earlier, stale count and an over-strong "frontier-first" line.)
