## Force-promote rationale

`--force-promote` was used. Two gate rows did not pass; each is overridden for a
documented reason, and neither is a load-bearing trust signal:

1. **`tag_age` (0.01 days < 3-day window).** This is the *expected* condition for
   a stable promotion that immediately follows a `/release`. The operator
   explicitly requested cutting a fresh rolling release and promoting it to stable
   in the same pass (capture today's working state as the rollback anchor). The
   soak window exists so the gate has time to *observe* a tag in the wild; here we
   are deliberately anchoring the just-shipped state, with the full local suite as
   the substitute evidence. Approved by the operator.

2. **`ci_green` = RED — but no CI job actually ran.** The GitHub Actions run on
   `64ca2e3` reports 6 failed checks, but every check failed in ~3 seconds with
   zero steps executed. The check annotation is explicit:

   > "The job was not started because recent account payments have failed or your
   > spending limit needs to be increased."

   This is an **account-billing halt**, not a code failure. No matrix leg ever
   checked out, installed, or ran a test — so the row's whole reason to exist
   (catching a matrix-leg failure invisible on the promoter's box) does not apply:
   there is no CI signal at all, in either direction. The condition is identical
   across the entire `master` history, including the `v0.10.0` baseline. The
   load-bearing evidence is therefore the **local suite**, which is green on
   `py3.13 · windows-latest` — the exact environment one of the "failed" CI legs
   would have run. Per the skill's own asymmetry note, a genuine RED (a real,
   accountable build break) would block; a *no-signal-dressed-as-RED* from a
   billing halt is the NO_SIGNAL case in disguise and is treated as advisory.

   **Follow-up:** once GitHub Actions billing is restored and CI produces a real
   green on a commit at or above `v0.11.0`, a subsequent stable promotion can be
   cut against a true CI signal. This evidence file records that the present
   promotion rests on the local suite, not CI.
