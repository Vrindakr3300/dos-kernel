Appendix D. Extended out-of-loop results

The body (§{{sec:payoff}}) carries the two headline live results — the write-admission gate (J = 10) and the race arbiter (J = 6). This appendix holds three further results that strengthen the existence claim without being the headline: a richer witness and the first measurement of what one propagated phantom costs (D.1); a frontier test that puts a frontier model in the verifier’s own seat against a ladder of non-forgeable rungs (D.2); and the compounding payoff across a branching fleet (D.3). All three read the same kind of byte the agent cannot author, and each carries its evidence-strength label in the same breath as its number.

D.1 A richer witness, and the first measurement of what propagation costs

tau2’s witness is a single bit, so a reader might ask whether the result is an artifact of that coarse signal. We re-aimed the identical gate at a second benchmark with a much richer witness: Agent-Diff (a write-heavy enterprise-API benchmark over Slack, Box, Linear, and Google Calendar), whose environment computes a full structured state-diff from database snapshots — the exact rows inserted, updated, deleted — and grades against a multi-assertion gold specification. Driving an agent live through Gemini on the 45-task test split, the detection result holds across the capability step a second time: flash over-claims on {{fact:ad_block_flash}} of its 35 confident writes ({{fact:ad_rate_flash}}), pro on {{fact:ad_block_pro}} of 31 ({{fact:ad_rate_pro}} — more, not less), and the gate blocks every one. The over-claims are partly model-specific (14 tasks over-claim on both models, the rest on only one), so the gate catches the deployed model’s blind spots, not a fixed set of impossible tasks. Detection generalizes off the coarse db_match bit to a structured diff and across two tiers.

Agent-Diff also lets us measure the thing J names but does not itself quantify: what does letting a phantom write propagate actually cost a downstream agent? We build the peer B handoff explicitly. For each blocked over-claim we run a second agent twice from the same fresh, correct environment, changing only what B is told it inherited: under believe B receives A’s claim verbatim; under adjudicate B receives the gate’s env-verified correction (“the prior write did not land — re-verify before acting”). The measured quantity is

ΔB = success(B | adjudicate) − success(B | believe)

over the over-claim slice. On flash, ΔB = {{fact:ad_deltab_flash}} (four flips, zero reverses): telling the peer the truth turned four phantom-induced failures into successes and never made one worse. On pro, ΔB = {{fact:ad_deltab_pro}} (two flips cancelled by two reverses; one reverse traced to a real gate bug — a blanket “treat the records as unchanged” over-corrected a partial over-claim in which one of two writes had landed. We fixed the gate for future runs but report the as-run number).

We lead with J, not ΔB, because they are a product and a lower bound on one of its terms, not two estimates of one thing. J counts the lies stopped at the root; ΔB measures the cost of letting one lie propagate one hop to the most forgiving consumer imaginable — an LLM peer that re-runs the task and can sometimes heal the missing write for free. So ΔB is a pessimistic floor: the gate's real value is blocks × depth × per-hop damage, and ΔB pins depth and consumer to their minimum. A consumer that does not re-check (a reward labeler, an auto-merge, a dashboard) recovers nothing. The depth term is what D.3 measures.

D.2 The frontier test: a ladder of rungs a frontier model cannot all climb

The cross-model results answer “does a stronger agent over-claim less” — it does not. The sharper version of the objection is that the latest model is smart enough that you no longer need a referee — either it stops over-claiming, or, as a judge, it sees through forgeries an older checker missed. We test the decisive second half: the referee here has two implementations, the deterministic floor that reads ground truth or a model asked to judge the same claim, so we put a frontier model in the judge seat and raced it against the floor. The model is Fable 5 (the newest at the time of writing), reached the way an operator would — straight through the Claude Code command line (claude -p), the model named only in an environment variable, not a line of model-specific code in the harness.

The test materializes nine artifacts as real, freshly authored git commits — seven forgeries plus two honest controls. Four forgeries touch the wrong files (an empty commit claiming the work; a keyword-stuffed subject; deleting the test that would have caught the gap; a long “production-ready” narration over a commit that did none of it). The other three are the adversary that matters — commits that touch the exact deliverable files the plan names but ship nothing behind them (empty stubs; code that does not import; a lookalike filename). Both channels see the same forgery-resistant evidence; the only asymmetry, stated up front so the fight is fair, is that the model judge also reads the agent’s narration. The floor is a ladder of three rungs: the commit subject (forgeable, the author types it); which files the commit touched (a commit cannot fake that, but it certifies presence, not behavior — a stub passes); and the OS running the test and reading the exit code (a 0 the agent cannot forge, because it did not run the process).

The result is not “the floor is perfect.” The presence rung — what most people picture by “check that the files landed” — is fooled {{fact:forge_presence_fa}} of {{fact:forge_floor_n}} by the path-touching stubs, the honest ceiling of any file-counting check. What sits above it is the point: the behavior rung holds at {{fact:forge_behavior_fa}} of {{fact:forge_floor_n}} and still confirms the one real landing, because it has the OS run the code and reads an exit code the agent cannot author (the forgeable subject rung is fooled on {{fact:forge_subject_fa}} of {{fact:forge_floor_n}}). The lesson is not “determinism wins” but which non-forgeable byte you condition belief on, and that the rung you must stand on rises with the forger’s sophistication.

The judge-side result is the more surprising one: the learned judge is exactly as good as the evidence the kernel computes for it. Given only narration, Fable abstains on the honest landing ({{fact:forge_fable_blind_ta}}/{{fact:forge_landings_n}}); given only which files were touched it still abstains ({{fact:forge_fable_ev_ta_blind_witness}}/{{fact:forge_landings_n}}) — it will not certify presence as completion. But given the OS test result — the byte the behavior rung reads — the same model confirms the work ({{fact:forge_fable_ev_ta}}/{{fact:forge_landings_n}}) and disagrees with every forgery ({{fact:forge_fable_ev_fa}}/{{fact:forge_floor_n}}). Left to its own reading the judge yields no value — gamed if forced to answer (35.2% false-accepts elsewhere), inert if allowed to abstain. The floor's value is that it computes the non-forgeable witness — including running the code — for free, every time, lifting both the deterministic verdict and the learned one. The stronger the model, the more the value concentrates in the byte it cannot author. (Full grid: docs/272.)

D.3 The payoff compounds: one block at the root keeps the whole downstream fleet clean

ΔB priced a single phantom propagating one hop. A real fleet is neither single nor shallow — a phantom is inherited again by the next agent, and a fleet branches — so the true quantity the gate protects is the whole set of downstream agents a root corruption reaches. We measured that set live, down a chain and across a fan-out tree, on the same tau2 airline witness with two arms (believe: each agent inherits the corrupt state; adjudicate: the gate blocked the root, so each inherits the gold state). The root corruption is one reservation R wrongly cancelled (injected; every downstream node runs live), and the witness is exact: is R still cancelled when this agent finishes?

Down a single chain the result is linear: under believe a depth-D chain leaves all D−1 downstream nodes corrupt, under adjudicate zero (the gate prevents 1, 2, and {{fact:cascade_chain_d4}} corrupt nodes at depths 2, 3, 4). The believe agents are instructive: each, handed a cancelled reservation, simply could not do its task and gave up — none restored R. So a capable peer re-checks and heals a narrated phantom (hence ΔB ≈ 0) but cannot re-check away a corrupt starting world; the state handoff is the one that compounds.

A chain understates a fleet, because fleets branch. A depth-D, fan-out-F dependency tree has FD leaves, all downstream of the root corruption. Under believe every leaf is corrupt (4 of 4 at depth 2, 8 of 8 at depth 3, F = 2); under adjudicate the single root block keeps every leaf clean. So the prevented-corruption payoff is FD — {{fact:cascade_tree_d2}} at depth 2, {{fact:cascade_tree_d3}} at depth 3 — against the chain’s linear D−1. FD here is breadth (FD agents each blocked by the one shared poison, not one corrupt value mutating into many): a poisoned shared resource fails every agent that depends on it, and that set grows FD as the fleet branches. This is the depth term ΔB pinned to one, now measured live. (Caveats: the root is injected, not waited for, and each node re-inherits the same root state rather than chaining onto the previous node’s output; it is an existence result for the compounding shape — one model, F = 2, depths 2–4 — not a calibrated constant. The fleet-of-one floor holds: at depth 0 the payoff is zero.)
