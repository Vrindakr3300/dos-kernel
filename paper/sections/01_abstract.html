Abstract

Checking one agent is easy: wrap it in “if the check fails, run it again.” That trick assumes the world holds still between the check and the act. In a fleet of concurrent agents it does not. A check reads true when agent A looks and false by the time A writes, because agent B changed the shared state in between (a time-of-check-to-time-of-use race{{cite:toctou}}, lifted onto world state). Agents act through high-level tool APIs that offer no transaction and no conditional write, so no re-run wrapper around A can close that race. It takes a referee between the agents that serializes their effects and reads only bytes the agents cannot author. We build one and run it live. The finding that surprised us is about where a non-forgeable verdict is worth spending. Hand it back to the agent that earned it (in-loop) and it is flat-to-harmful: a capable agent re-checks and heals its own phantom, so every active fix we measured (warn, rewind, inject a cure) only disturbs a run that would have passed. The lone in-loop survivor is the negative action, give up correctly (zero false-halts across 1,634 winners, ~11% compute saved). Hand the same verdict to anyone else — the rest of the fleet — and it pays, on both of the ways a fleet fails. Off the environment’s own database hash (a byte no agent wrote), an arbiter serializes eight pairs of racing agents to prevent all 6 would-be lost-update clobbers, and a write-admission gate blocks 10 genuine over-claims across 120 clean tasks while admitting every correct write. The block rate is an identical 8.3% on a mid and a strong model, so the signal does not fade at the stronger tier. A third consumer is not an agent at all. Each blocked over-claim is a poison reward label a self-judged RL loop would bank as a win, and scoring the reward against the byte the model cannot author — the verifiable-reward (RLVR{{cite:rlvr}}) discipline — purges every one (label purity 60% → 100%). One non-forgeable check, three consumers: a monitorability primitive for fleets and the loops that train them, whose value turns on where it is spent.

Introduction

Give an agent a real task — reconcile an inventory across two SaaS apps, send a survey to every customer — and it works for many turns, then announces success. Often the announcement is a lie the agent believes. It wrote “All customers have now received their surveys!” and the surveys never went out. This is silent failure: the run looks finished, the world is still broken, and nothing in the agent’s own words tells you which is true. For one agent the skeptic reaches for a tidy answer, and at the easy hop it is right, so we grant it up front: do not trust the “done,” check the work, re-run if the check fails. We grant that loop — while noting two parts of it are real work the phrase “just check the result” hides: a check the agent cannot forge (not “re-read your own result”), and sensing fast enough to gate every turn (§{{sec:positioning.wrapper}}). But the claim we rest the paper on is what a re-run wrapper genuinely cannot reach. The instant you have a fleet — many agents working concurrently against the same database, the same tickets — the assumption breaks: a check reads true when agent A looks and false by the time A writes, because agent B changed the shared state in the gap (a time-of-check-to-time-of-use race{{cite:toctou}}, lifted from files onto world state). The textbook cure — fuse check and write into one atomic step — lives in the store, and the high-level tool APIs agents act through (cancel a reservation, file a ticket) expose only unconditional writes. So no re-run wrapper around A can close the race. It takes a referee between the agents that supplies the missing serialization itself and trusts none of their narration. That referee, and the one surprising place its verdict is worth spending, is this paper.

The trust rule under everything is one test: read what the agent says, or read bytes the agent could not have written. We call the second byte-clean — a check whose byte-author is not the agent under judgement (an environment’s tool result qualifies; the agent’s own prose does not). Reading the agent’s own bytes is the mirror-verifier trap: it measures whether the story is internally consistent, not whether it matches the world, and collapses the moment the model stops emitting the give-away phrase you trained on. (A plain-language walk-through of every term is in Appendix A; the machinery in Appendix B.)

The result is not a detector but a place to spend one’s verdict. Hold a check the agent cannot forge fixed, and ask only where. Hand it in-loop, back to the agent that earned it, and it is worthless: every active fix we measured was flat or worse, because injecting any turn perturbs runs that would have passed. Hand it out-of-loop, to the rest of the fleet, and it pays on the two distinct ways a fleet fails — a claim (one agent announces a write that did not land, and a peer inherits it as truth) and a race (two agents touch the same state and one overwrites the other). In §{{sec:payoff}} we build one out-of-loop referee reading the environment’s own database hash, run it live on tau2-bench{{cite:tau2bench}} through Gemini, and it blocks 10 real over-claims across two models at an identical 8.3% rate while serializing two live agents across eight conflict pairs to prevent all six would-be clobbers — both off ground truth, neither touching the judged agent. A third consumer is the loop that trains the next model: each blocked over-claim is a poison reward label a self-judged RL loop would bank as a win, and scoring the reward against the same byte (RLVR{{cite:rlvr}} in the strict sense) purges every one. The rest of the paper is the boundary that forces this design. One distinction throughout: J counts real corruptions a non-forgeable witness blocked, never a downstream-outcome delta; where we measure the latter (ΔB) it is a fragile floor, and we say so in the same breath.

Why this matters past this paper. A verdict the agent cannot forge or train away is the load-bearing primitive under three problems a frontier lab already cares about, each touched with a measured result. Scalable oversight{{cite:bowman}}: when work outruns a human’s line-by-line check{{cite:metr}}, the only oversight that survives is rooted in evidence the model did not author (the byte-clean rule and its rungs W0–W3, §{{sec:boundary}}). Reward hacking / monitorability{{cite:baker}}: a self-judged loop rewards the appearance of success, and a label non-forgeable by narration is the one signal optimization pressure cannot launder — we show it purges real poison reward labels off ground truth. Multi-agent reliability: the race no single-agent wrapper can reach needs a referee between agents, and we build one and measure the corruptions it prevents live. The method borrows a minimal-trusted-kernel framing{{cite:anderson}}, three distributed-systems hazards (TOCTOU{{cite:toctou}}, lost-update{{cite:bernstein}}, write-ahead recovery{{cite:aries}}), and the SPRT{{cite:wald}}/CUSUM{{cite:page}} lineage of the give-up gate — and it states the strength of every claim in the same breath as the claim, never dressing a rate as a payoff or a frozen replay as a live result.
